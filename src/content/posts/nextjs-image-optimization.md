## 문제: next/Image가 느리다

사내에서 `next/image` 패키지를 사용해 이미지를 최적화하는 중에, **Time To View**가 너무 느린 문제가 생겼다. 평균 1~2초, 심하면 3초 이상이었다.

응답 헤더를 보니 `X-NextJS-Cache`에서 캐시 HIT 여부를 담아 보내는데, 캐시 HIT 비중이 지나치게 적었다. 일부 이미지는 계속 MISS가 나고 있었다.

---

## next/image의 동작 원리

요청된 이미지를 **Squoosh**(또는 sharp) 라이브러리를 통해 WebP로 경량화한 이미지를 반환하고, 이를 Next.js 내부 캐시 폴더(`<distDir>/cache/images`)에 저장한다.

사용자가 지정한 TTL(기본 60초, `minimumCacheTTL` 설정 가능) 이 만료된 리소스는 STALE 처리된다. 유효기간이 만료된 리소스를 재요청하면 캐싱된 이미지를 반환하고, 즉시 새 이미지를 받아 리사이징해 캐시를 업데이트한다.

모든 이미지가 대상은 아니다. SVG나 GIF 같은 이미지는 리사이징 대상에서 제외된다.

---

## 왜 느린가

**캐싱이 안 된 이미지는 오히려 더 느리다.** 리사이징 과정을 거쳐야 하기 때문이다.

**리사이징 요청이 몰리면 서버에 부하가 걸린다.** 캐싱되지 않은 이미지가 많으면 그만큼 리사이징을 많이 하므로 Next.js 서버의 응답이 느려진다.

**배포 시마다 캐시가 사라진다.** 캐싱된 이미지는 Next.js 서버에 저장되기 때문에 재배포하면 기존 캐시가 사라진다. 매 배포마다 이미지를 다시 리사이징해야 한다.

---

## 해결 방법

### 방법 1: unoptimized 옵션 활용

불필요한 리사이징을 줄이는 방법이다. 하지만 어떤 이미지에 리사이징을 하지 말아야 할지 개발자 단에서 판단해야 하고, 모든 이미지에 적용하면 `next/image`를 쓰는 이유가 사라진다.

### 방법 2: Squoosh 대신 sharp 설치

```bash
npm install sharp
```

sharp는 C++ 기반으로 동작해 리사이징 속도와 리소스 면에서 Squoosh보다 효율적이다. Next.js도 sharp 사용을 권장한다. 하지만 근본적으로 리사이징 처리를 Next.js 서버에서 해야 한다는 사실은 변하지 않는다.

### 방법 3: On-The-Fly (외부 CDN 위임)

이미지 리사이징과 캐싱 자체를 Next.js에게 위임하지 않고 **외부에 맡기는 방법**이다.

AWS CloudFront(CDN)와 AWS Lambda를 조합한다. 이미지에 대한 요청이 왔을 때 캐싱 여부를 판별하고, 캐싱이 안 됐다면 Lambda에서 리사이징한 이미지를 CDN에 캐싱하도록 한다.

---

## On-the-fly 동작 방식

```
1. CloudFront에 이미지 요청
2. 캐싱 여부 확인
   ├── 캐시 HIT → 캐싱된 이미지 즉시 반환
   └── 캐시 MISS → S3에서 원본 이미지 가져옴
3. Origin Response 단계에 Lambda 트리거
4. Lambda에서 sharp로 이미지 리사이징
5. 리사이징된 이미지를 CDN에 캐싱
6. 응답 반환
```

CloudFront에서는 `Origin Response` 단계에 Lambda를 트리거한다. Lambda에서는 응답으로 반환된 이미지를 sharp로 리사이징하고 CDN에 저장한다. 재방문 시에는 CDN에서 바로 리사이징된 이미지를 반환한다.

CloudFront와 Lambda를 같은 Region에서 실행하려면 **Lambda@Edge**를 사용한다. 사용자에게 더 가까운 엣지 로케이션에서 Lambda를 실행할 수 있어 레이턴시를 줄인다.

---

## 고려해야 할 점

- CDN 사용은 비용이 드는 일이므로 인프라 팀과의 논의를 통해 도입을 결정해야 한다
- 모든 이미지를 전부 캐싱하는 것도 좋은 방법이 아니므로 적절한 캐시 무효화 정책이 필요하다
- 초기 요청 시 리사이징 과정을 타기 때문에 next/image보다 빠르지만, 첫 요청의 TTV는 살짝 느려질 수 있다
- 배포 후에도 CDN 캐시는 유지되므로, 이미지 업데이트 시 캐시 무효화 처리를 별도로 해줘야 한다
